## 三、代码缺陷与改进建议

> 以下为代码审查中发现的问题，已全部修复（2026-06-21）。

### BUG-01（中等）✅已修复：FSRS `elapsed_days` 始终为 0

**位置**：`src/lib/db.ts` calculateNextReview()

**问题**：数据库中存储的 `elapsed_days` 在每次 review 后被重置为 0，导致 FSRS 算法无法判断复习准时/延迟/提前。

**修复方案**：在前端 `calculateNextReview` 调用 Rust 前，根据 `next_review_at` 和 `scheduled_days` 反推上次复习日期，计算真实的天数差后传给 FSRS。

**验证测试**：SM-18（FSRS 参数持久化）、RV-05/RV-06/RV-07（评分后间隔变化）

### BUG-02（低）✅已修复：CSV 导入不支持引号内含逗号的字段

**位置**：`src/pages/VocabularyPage.tsx` handleFileChange()

**修复方案**：实现 RFC 4180 兼容的 `parseCsvLine()` 函数，支持引号字段内逗号、转义双引号、自动检测 Tab/逗号分隔符。

**验证测试**：BD-14（含逗号字段导入）、VP-16 至 VP-21（所有 CSV 导入测试）

### BUG-03（低）✅已修复：Anki 导出不转义 Tab/换行符

**位置**：`src-tauri/src/commands/mod.rs` export_words_anki()

**修复方案**：导出前对 phonetic、definition、notes 字段中的 `\t` 和 `\n` 替换为空格。

**验证测试**：VP-27（Anki 导出测试）

### BUG-04（低）✅已修复：HistoryDetailPage 未懒加载 KnowledgeGraph

**位置**：`src/pages/HistoryDetailPage.tsx` 第 18 行

**修复方案**：改为 `lazy()` 懒加载配合 `<Suspense>` fallback，与 ReadingPage 保持一致。

**验证测试**：HI-08（阅读详情含图谱时正常加载）

### BUG-05（低）✅已修复：TTS 测试静默失败

**位置**：`src/pages/SettingsPage.tsx` handleTestTTS()

**修复方案**：添加 `ttsTestError` 状态，失败时在按钮旁显示红色错误提示。

**验证测试**：SE-08（TTS 测试成功）、SE-09（TTS 测试失败时显示错误）

### BUG-06（低）✅已修复：词汇补全失败无用户反馈

**位置**：`src/pages/VocabularyPage.tsx` handleEnrich()

**修复方案**：失败时通过 `showMessage` 显示“补全失败，请检查网络连接”提示。

**验证测试**：VP-08（单个补全失败降级）

### BUG-04a（低）✅已修复：CorrectPage 历史写入静默失败

**位置**：`src/pages/CorrectPage.tsx` handleCorrect() → onDone

**修复方案**：添加 `saveError` 状态，通过 `addHistorySafe` 的 onError 回调设置警告，在 UI 显示琥珀色横幅。

**验证测试**：WC-13（持久化验证）

### BUG-04b（低）✅已修复：ReadingPage 图谱失败无反馈

**位置**：`src/pages/ReadingPage.tsx` useGraphData() 返回值

**修复方案**：解构 `graphError` 并在图谱失败且无数据时显示琥珀色警告横幅。

**验证测试**：RP-10（知识图谱生成）、RP-11（图谱 AbortSignal）

### ISSUE-07：旧测试计划中 Sidebar 行为描述已过时

**位置**：`docs/test-plan-260602.md` SB-06

**问题**：旧计划描述 "Sidebar 仅在应用启动时加载一次数据"，但代码中 Sidebar 已在每次导航时重新获取数据。

**状态**：已在新测试计划中更新为正确描述（SB-06）。

### ISSUE-08：旧测试计划中复习算法描述已过时

**问题**：旧计划描述简单的“间隔翻倍”算法，但代码已迁移至 FSRS 算法。
