# Raven — 英语学习助手 设计文档

## 概述

Raven 是一款基于 Tauri + React 的桌面英语学习助手，提供翻译、纠正、精读三大核心功能，辅以生词本、历史记录和知识图谱，帮助用户通过 LLM 驱动的深度分析提升英语能力。

## 技术栈

| 层级 | 选型 | 说明 |
|------|------|------|
| 桌面框架 | Tauri | 轻量级桌面应用，Rust 后端 + WebView 前端 |
| 前端框架 | React + TypeScript | 组件化 UI 开发 |
| UI 组件库 | shadcn/ui | 可定制的现代 UI 组件 |
| 存储 | SQLite（Tauri 内嵌） | 本地持久化，存储生词本、历史记录、设置 |
| LLM | OpenAI 兼容 API | 前端 JS 调用，支持多模型（Qwen、月之暗面等） |
| 知识图谱 | Cytoscape.js | 精读页面的词汇网络和概念关系可视化 |

## LLM 抽象层

设计一个 Agent 层对 LLM 进行抽象，不绑定单一模型：

- 统一接口：基于 OpenAI Chat Completions API 规范
- 配置驱动：通过设置页面配置 API key、base URL、模型名称
- 多模型支持：用户可保存多个模型配置，按场景切换
- 流式响应：前端通过 fetch + ReadableStream 处理 SSE 流
- API key 安全：存储在 SQLite 中，通过 Tauri command 注入前端

## 功能模块

### 1. 翻译

**布局**：统一堆叠式，LLM 自动识别输入语言，给出最佳翻译。

**输入区**：文本输入框 + 翻译按钮

**结果展示**（自上而下）：

#### 1.1 翻译结果
- LLM 给出的最佳翻译文本

#### 1.2 用词分析
对翻译过程中的关键短语进行分析，每个短语包含：
- 英文原文 + 中文对应
- 按需标注（三种标注可叠加）：
  - **📌 固定搭配**：搭配结构讲解 + 同类替换词
  - **🌍 地道表述**：与直译的差异 + 为何需要转化
  - **🏷️ 行业术语**：领域背景 + 标准译法 + 不可意译说明

#### 1.3 句子结构拆解
- 原句标注（主干、修饰成分高亮）
- 主干提取
- 修饰成分说明（从句类型、分词短语、介词短语等）
- 句型归纳

#### 1.4 翻译技巧提示
- 语序调整
- 隐喻转化
- 关联词转换
- 程度词选择
- 转述处理等

---

### 2. 纠正

**布局**：报告式堆叠布局

**输入区**：文本输入框 + 开始纠正按钮

**结果展示**（自上而下）：

#### 2.1 语法纠正
- 错误标注（红色高亮）
- 正确修正（绿色高亮）
- 语法规则说明

#### 2.2 词汇纠正
- 用词不当标注
- 建议替换词 + 语境适配说明

#### 2.3 地道文化表达纠正
- 中式英语表达 → 地道英文表达
- 文化语境说明

#### 2.4 句型分析
- 长难句结构拆解
- 主干提取 + 修饰成分标注
- 句型归纳（主谓宾、主系表、复合句等）

#### 2.5 纠正总结
- 问题统计（语法 / 词汇 / 表达各几处）
- 共性问题归纳
- 学习建议

---

### 3. 精读

**布局**：左右分栏

#### 左侧 — 原文区
- 完整英文原文展示
- 单词点击标注（点击生词高亮，可添加到生词本）
- 长难句高亮标记

#### 右侧 — 分析结果
- 词汇解析（音标、释义、搭配）
- 句法分析（句子成分标注）
- 文化背景说明
- 知识图谱（Cytoscape.js 可视化）

#### 知识图谱
使用 Cytoscape.js 实现：
- 节点：词汇、概念、实体
- 边：同义、反义、搭配、上下位等关系
- 布局：力导向布局，支持缩放、拖拽、点击展开
- 交互：点击节点高亮关联节点和原文对应位置

### 4. 生词本

- 词汇列表展示
- **分级标记**：
  - CET-4（英语四级）
  - CET-6（英语六级）
  - TEM-4（英专四级）
  - TEM-8（英专八级）
- 来源记录（来自哪篇文章的精读）
- 添加时间
- 复习状态
- 支持搜索和筛选

### 5. 历史记录

- 按功能分类（翻译 / 纠正 / 精读）
- 时间线展示
- 支持搜索
- 点击可回看完整结果

### 6. 设置

- LLM 配置：
  - API Key
  - Base URL
  - 模型名称
  - 保存多个模型配置
- 界面设置：
  - 语言（中文 / 英文）
  - 主题（亮色 / 暗色）
- 数据管理：
  - 导出 / 导入生词本
  - 清除历史记录

## 数据模型（SQLite）

### words（生词本）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| word | TEXT | 单词 |
| definition | TEXT | 释义 |
| level | TEXT | 分级：CET-4 / CET-6 / TEM-4 / TEM-8 |
| source_id | INTEGER | 来源记录 ID |
| phonetic | TEXT | 音标 |
| notes | TEXT | 用户备注 |
| created_at | DATETIME | 添加时间 |
| review_status | TEXT | 复习状态 |

### history（历史记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| type | TEXT | 类型：translate / correct / reading |
| input_text | TEXT | 输入文本 |
| result | TEXT | 结果 JSON |
| created_at | DATETIME | 创建时间 |

### settings（设置）
| 字段 | 类型 | 说明 |
|------|------|------|
| key | TEXT PK | 设置键 |
| value | TEXT | 设置值 |

### models（LLM 模型配置）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK | 自增主键 |
| name | TEXT | 配置名称 |
| api_key | TEXT | API Key |
| base_url | TEXT | API 地址 |
| model_name | TEXT | 模型名称 |
| is_default | BOOLEAN | 是否默认 |

## 交互流程

### 翻译流程
1. 用户输入文本（中/英均可）
2. 点击「翻译」
3. LLM 自动识别语言，生成翻译 + 用词分析 + 句子拆解 + 技巧提示
4. 结果以堆叠式卡片展示
5. 用户可将生词添加到生词本

### 纠正流程
1. 用户输入英文文本
2. 点击「开始纠正」
3. LLM 生成纠正报告（语法 + 词汇 + 地道表达 + 句型分析 + 总结）
4. 结果以报告式卡片展示

### 精读流程
1. 用户输入或粘贴英文文章
2. 左侧展示原文
3. 右侧展示 LLM 分析结果
4. 用户点击单词可标注并添加到生词本
5. 知识图谱展示词汇和概念的关系网络
