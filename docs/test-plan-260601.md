# Raven 测试计划

> 版本：v1.5.0 · 日期：2026-06-01 · 测试环境：Windows 11 + Tauri v2

---

## 一、测试范围

本测试计划覆盖 Raven v1.5.0 的全部功能，包括 11 个页面、3 个自定义 hook、6 张数据库表、9 个 LLM 集成点、5 个 TTS 使用场景。

### 测试类型

| 类型 | 覆盖范围 | 执行方式 |
|------|----------|----------|
| 功能测试 | 所有用户交互和业务逻辑 | 手动 |
| 状态机测试 | 3 个 phase machine + task status + word review | 手动 |
| LLM 集成测试 | 9 个 prompt + 流式响应 + JSON 解析 | 手动（需 API Key） |
| TTS 集成测试 | 5 个 TTS 使用场景 | 手动（需 TTS API Key） |
| 边界测试 | 空输入、网络断开、并发操作 | 手动 |
| 回归测试 | 新功能不影响已有功能 | 手动 |

---

## 二、前置条件

### 环境准备

- [ ] Windows 10/11 系统
- [ ] 应用已安装并可启动
- [ ] OpenAI 兼容 API Key 已准备（或 DeepSeek API Key）
- [ ] TTS API Key 已准备（可与 LLM 共用）
- [ ] 网络连接正常

### 测试数据准备

- [ ] 至少 3 篇英文作文（用于写作批改测试）
- [ ] 至少 2 篇英文文章（用于阅读精读测试）
- [ ] 测试用 CSV 文件（word,phonetic,definition,level 格式，10 行）
- [ ] 空 CSV 文件（用于边界测试）
- [ ] 含重复词的 CSV 文件（用于去重测试）

## 三、功能测试用例

### 3.1 新用户引导（OnboardingDialog）

**OF-01** 首次启动显示引导对话框
- 前置：models 表为空，onboarding_done 未设置
- 操作：启动应用
- 预期：显示 4 步引导对话框，第一步显示功能介绍

**OF-02** 引导步骤导航
- 操作：点击"开始配置"→ 填写 API 信息 → 测试连接 → 下一步 → 下一步 → 开始使用
- 预期：每步正确切换，模型保存到数据库，onboarding_done 设为 "true"

**OF-03** API 连接测试
- 操作：输入正确的 API Key 和 URL，点击"测试连接"
- 预期：显示"连接成功"绿色提示

**OF-04** API 连接测试失败
- 操作：输入错误的 API Key，点击"测试连接"
- 预期：显示错误信息（HTTP 状态码 + 错误文本），"下一步"按钮禁用

**OF-05** 跳过引导
- 操作：在任意步骤点击"跳过"
- 预期：对话框关闭，onboarding_done 设为 "true"，不再弹出

**OF-06** 已配置用户不显示引导
- 前置：models 表有记录
- 操作：启动应用
- 预期：不显示引导对话框

**OF-07** Escape 键不关闭引导
- 操作：按 Escape 键
- 预期：对话框不关闭（必须显式跳过或完成）

**WC-11** JSON 解析失败降级

## 十、已知限制

1. **TTS 播放依赖外部 API**：无网络时 TTS 功能不可用
2. **LLM 输出不可控**：JSON 解析依赖三级 fallback，极端情况下可能降级为原始文本
3. **间隔重复算法简单**：相比 Anki 的 SM-2，Raven 的算法缺少 Ease Factor 调整
4. **SpeedTrainerPage 未使用 useAudioPlayer**：循环逻辑复杂，有意保持独立实现
5. **ReadingPage 语言检测消耗一次 LLM 调用**：可用正则预过滤减少 API 调用
